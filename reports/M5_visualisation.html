Intermittency — % Zero-Sales Days
68.2% of all (series × day) combinations are zero. This is a defining challenge — standard regression losses are poorly suited.

Why it matters:
Many items only sell on specific days (e.g. seasonal, event-driven). Using Tweedie or Poisson loss instead of MSE/MAE is strongly advised for intermittent series.
RMSSE handles this: the denominator is computed only from the period after the first non-zero sale, preventing division-by-zero for entirely-zero series.
05 Metric — WRMSSE (Weighted Root Mean Squared Scaled Error)
Step 1 — RMSSE per series
# For a single aggregated time series i: RMSSE_i = sqrt( Numerator / Denominator ) Numerator = mean( (Y_true[t] - Y_pred[t])² ) over 28 forecast days → mean squared forecast error Denominator = mean( (Y[t] - Y[t-1])² ) over active training period (starts from first non-zero day) → naive random-walk in-sample MSE ────────────────────────────────── RMSSE = 1.0 → same as naive baseline RMSSE < 1.0 → better than naive ✓ RMSSE > 1.0 → worse than naive ✗
Step 2 — Weights within each level
# Weight = revenue share in last 28 training days w(i, level) = Σ( price(i,t) × units(i,t) ) # last 28d ──────────────────────────── Σⱼ( price(j,t) × units(j,t) ) # all series in level Example: within Level 4 (Category), FOODS weight ≈ 56.9% (highest revenue) HOUSEHOLD weight ≈ 30.3% HOBBIES weight ≈ 12.8% (lowest revenue) → Errors on high-revenue items penalised more → FOODS accuracy matters most at every level
Step 3 — Final WRMSSE score
WRMSSE = (1/12) × Σₗ [ Σᵢ∈level_l ( w(i,l) × RMSSE(i,l) ) ] = equal-weight average across 12 hierarchy levels = each level contributes exactly 1/12 to final score Winning scores: ~0.5 – 0.6 Naive baseline: 1.0 Random: >> 1.0

External drivers
